# Product

## Register

product

## Users

主要用户是公司产品运营人员。他们在日常工作中需要围绕产品页面、活动页、关键词、内容结构和页面转化目标，快速获得可执行的 SEO 建议。

本项目也是 Agent 应用开发学习项目，但首页和核心体验不应表现成学习演示页。面向用户的界面要优先服务真实运营工作流，学习与工程复盘放在文档、日志和代码结构中体现。

## Product Purpose

AI SEO Agent 是一个任务型对话工作台，帮助产品运营用自然语言描述页面、产品、关键词或优化目标，并获得 SEO 策略、页面优化建议、关键词想法、内容结构建议和基础检查结果。

产品成功标准不是一次性做成复杂商业系统，而是先形成稳定、清晰、可用的小型 Agent 产品闭环：用户输入 SEO 任务，后端组织上下文并调用模型，必要时结合本地工具检查，前端以舒服、可信、低干扰的方式展示结果。

## Brand Personality

时尚简约、克制舒适、专业清晰。

整体气质可以参考 Claude Code 给人的现代工具感：安静、聚焦、好读、让人愿意长时间使用。但它不应直接复制 Claude 的视觉语言，而应保留 AI SEO Agent 自己的产品边界、信息结构和业务识别度。

## Anti-references

不要直接照搬 Claude 或其他现成 AI 产品的外观。

不要做成复杂、沉重、难读的后台系统，也不要出现死气沉沉的传统 menu 侧边栏体验。避免过度装饰、颜色偏重或不搭配、视觉元素抢走任务注意力、组件样式堆叠导致页面看起来复杂。

不要为了组件库统一牺牲业务布局。强定制导航、业务卡片和会话区域应优先保持清晰可读的业务样式，复杂交互再交给基础组件库。

## Design Principles

1. 任务优先，界面退后：运营人员应先看见输入、对话和结果，而不是装饰性的品牌表达。
2. 简约但不冷漠：页面要安静、舒服、有现代感，但不能像传统后台一样沉闷。
3. 对话就是工作台：会话、上下文、错误和结果状态都要清楚可追踪，服务真实 SEO 工作推进。
4. 克制使用组件库：基础组件服务复杂交互，业务结构保持可读、稳定、容易维护。
5. 工程学习退居幕后：Agent 概念学习沉淀在文档、日志和代码结构中，不让首页像教学演示或调试台。

## Accessibility & Inclusion

暂无额外指定的合规要求。后续设计和实现默认以清晰可读、键盘可用、焦点状态明确、正文对比度达到 WCAG AA、支持 `prefers-reduced-motion` 为基本底线。
