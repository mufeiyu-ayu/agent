# 更新 Agent 学习日志

## 概述

维护项目内的 `docs/learning-log.md`，让 Agent 应用开发学习过程可追踪、可复盘。这个 skill 关注学习记录，不替代代码实现、测试或提交。

## 默认流程

1. 先确认当前工作目录是否为本项目根目录或其子目录；如果不是，先定位当前项目根目录和 `docs/learning-log.md`。
2. 读取项目 `AGENTS.md`、`CLAUDE.md`、已有 `docs/learning-log.md`、`git status --short`，并按需要读取本次涉及的关键文件。
3. 判断本次应该新增哪类记录：
   - 阶段性功能完成
   - Agent 概念学习或澄清
   - 错误排查和修复
   - 项目结构或学习流程调整
4. 更新 `docs/learning-log.md`：
   - 如果文件不存在，创建包含“学习路线看板”“日志记录”“记录规则”的最小模板。
   - 每次有实质学习进展时，在“日志记录”表格追加一行。
   - 如果学习阶段状态变化，同步更新“学习路线看板”。
5. 更新后做轻量自审，确认记录具体、真实、可复盘，没有泄露敏感信息。

## 表格字段

“日志记录”表格使用这些字段：

- `日期`：使用 `YYYY-MM-DD`。
- `阶段`：写学习阶段，不写空泛标题。
- `本次学习 / 实现`：说明实际完成了什么。
- `Agent 关键概念`：写本次真正涉及的 Agent 概念。
- `关键入口`：写项目相对路径；可包含本项目 `.claude/skills` 或 `.codex/skills` 下的 skill 路径。
- `验证 / 结果`：写执行过的命令、观察到的结果或未验证原因。
- `复盘`：一句话说明这次学习的关键理解。
- `下一步`：写一个具体、可执行的下一步。

## 记录原则

- 用简体中文记录。
- 只记录真实发生的事实，不替用户补不存在的学习结论。
- 优先写“概念如何落到工程实现”，不要写空泛心得。
- 小任务可以一行记录；复杂任务可以在表格外新增简短小节，但不要把日志写成日报。
- 不记录 API Key、token、密码、私有服务密钥、完整报错中的敏感字段。
- 如果本次只是普通问答，没有阶段性学习或项目变化，可以先询问用户是否需要记录；如果用户明确说每次都记录，则追加简短记录。

## 推荐记录粒度

一条记录应能回答这些问题：

- 今天推进到了哪个阶段？
- 学到了哪个 Agent 概念？
- 这个概念对应到哪个代码入口？
- 有哪些命令或结果能证明它可用？
- 下一步应该练什么？

## 输出方式

完成更新后，简短告诉用户新增或修改了哪条记录，并给出 `docs/learning-log.md` 的位置。只有在用户要求时才粘贴完整日志内容。
